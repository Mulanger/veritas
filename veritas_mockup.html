VERDICT LANGUAGE · DESIGN RULES

Likely authentic

Green reserved for when provenance actually confirms something, not for "we didn't detect a fake." Absence of evidence is never authenticity.

Uncertain

Not a failure. Amber is calm, not alarming. The card explains why — low quality, disagreement, missing signal — and suggests a next step.

Likely synthetic

Red only when ensemble agrees. Every red verdict ships with at least two independent reasons. Never red on a single detector's vote.

Confidence is a range

Display 78–94%, never 86%. False precision is the fastest way to lose the user's trust when we're wrong.

Never color alone

Every verdict carries an icon, a code, and a label. ~8% of users don't reliably distinguish red/green. The layout must read the same in grayscale.

Return path matters

Every verdict screen has a single dominant action and a "Done" that sends the user back to the app they came from. Veritas is an interrupt, not a destination.
